# ServiceNow Shared Folder Access Lab

## Overview

This lab demonstrates how to use ServiceNow to resolve a user access issue involving a shared network folder.
A user contacted the help desk because they were unable to access a shared company folder.

The issue was documented and resolved through the ServiceNow ticketing system.

---

## Scenario

A user attempted to open a shared network folder but received an access denied error.

The user contacted the IT help desk for assistance.

---

## Tools Used

* ServiceNow (Incident Management)
* Windows File Sharing
* Access Control / Permissions

---

## Ticket Details

Issue Type: Access Issue
Category: File Share Access
Priority: Medium

User reported they were unable to open a shared folder used by their department.

---

## Troubleshooting Steps

1. Created an incident ticket in ServiceNow.
2. Gathered details from the user about the folder path.
3. Verified the user’s permissions for the shared folder.
4. Identified that the user was missing the correct access permissions.
5. Updated access permissions for the shared folder.

---

## Resolution

The user was granted the appropriate access permissions to the shared folder.

The user confirmed they were able to successfully access the folder after the change.

The incident ticket was then marked as **Resolved** and **Closed**.
